### 1.2 電腦的五大單元

所有電腦（包括你的手機、筆電、甚至本專案的 LM3S6965 微控制器）都由五大單元組成：

```
┌─────────────────────────────────────────────────────────────┐
│                      電腦五大單元                              │
│                                                               │
│   ┌──────────┐         ┌──────────────┐         ┌─────────┐ │
│   │  輸入單元  │   →    │   控制單元     │   ←→  │ 記憶單元  │ │
│   │          │         │   (Control)   │         │(Memory) │ │
│   │ 鍵盤/感測器│         │               │         │         │ │
│   └──────────┘         │  負責協調指揮   │         │ 資料儲存  │ │
│                        └──────────────┘         └─────────┘ │
│   ┌──────────┐               ↕                               │
│   │  輸出單元  │         ┌──────────────┐                      │
│   │          │   ←     │   運算單元     │                      │
│   │螢幕/LED燈 │         │     (ALU)     │                      │
│   └──────────┘         │               │                      │
│                        │  執行加減乘除   │                      │
│                        │  邏輯運算      │                      │
│                        └──────────────┘                      │
│                                                               │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  說明：                                                 │   │
│  │  • 輸入單元：接收外界資料（鍵盤、滑鼠、感測器）            │   │
│  │  • 輸出單元：顯示結果（螢幕、LED、馬達）                  │   │
│  │  • 記憶單元：儲存程式和資料（RAM、Flash）                │   │
│  │  • 控制單元：協調各單元工作（CPU 的一部分）               │   │
│  │  • 運算單元（ALU）：執行計算（CPU 的一部分）              │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```


## 2. 數字系統：電腦的語言

### 2.1 為什麼電腦用二進位？

電腦內部使用「電」來工作，而電只有兩種狀態：

```
┌────────────────────────────────┐
│  電路的兩種狀態                   │
│                                 │
│  狀態 1：有電（高電位）            │
│  ┌────┐                         │
│  │ 💡 │ ← 燈亮                  │
│  └────┘                         │
│  代表：1                         │
│                                 │
│  狀態 2：沒電（低電位）            │
│  ┌────┐                         │
│  │    │ ← 燈滅                  │
│  └────┘                         │
│  代表：0                         │
│                                 │
└────────────────────────────────┘
```

因為只有兩種狀態，所以電腦使用「二進位」（Binary）數字系統。

### 2.3 位元（Bit）與位元組（Byte）

```
┌────────────────────────────────────────────────┐
│  1 位元 (Bit) = 1 個二進位數字                    │
│  ─────────────────────────                      │
│                                                 │
│    0  或  1                                     │
│    ↑                                            │
│   1 bit                                         │
│                                                 │
├────────────────────────────────────────────────┤
│  1 位元組 (Byte) = 8 個位元                      │
│  ─────────────────────────                      │
│                                                 │
│   ┌───┬───┬───┬───┬───┬───┬───┬───┐            │
│   │ 1 │ 0 │ 1 │ 1 │ 0 │ 0 │ 1 │ 0 │            │
│   └───┴───┴───┴───┴───┴───┴───┴───┘            │
│    ↑                               ↑            │
│   最高位元 (MSB)           最低位元 (LSB)         │
│                                                 │
│   1 Byte = 8 bits                               │
│   範圍：0 ~ 255 (2⁸ - 1)                        │
│                                                 │
├────────────────────────────────────────────────┤
│  1 字組 (Word) = 2 個位元組                      │
│  ─────────────────────────                      │
│                                                 │
│   16 bits = 2 Bytes                             │
│   範圍：0 ~ 65535 (2¹⁶ - 1)                     │
│                                                 │
├────────────────────────────────────────────────┤
│  1 雙字組 (Double Word) = 4 個位元組             │
│  ───────────────────────────────                │
│                                                 │
│   32 bits = 4 Bytes                             │
│   範圍：0 ~ 4,294,967,295 (2³² - 1)             │
│                                                 │
│   ← ARM Cortex-M3 是 32-bit 處理器              │
│                                                 │
└────────────────────────────────────────────────┘
```

#### 記憶體容量單位

```
1 Byte (B)   = 8 bits
1 Kilobyte (KB)  = 1,024 Bytes    ≈ 1 千 Bytes
1 Megabyte (MB)  = 1,024 KB       ≈ 1 百萬 Bytes
1 Gigabyte (GB)  = 1,024 MB       ≈ 10 億 Bytes
1 Terabyte (TB)  = 1,024 GB       ≈ 1 兆 Bytes

範例：
LM3S6965 微控制器（本專案）:
  - RAM：64 KB  = 65,536 Bytes
  - Flash：256 KB = 262,144 Bytes

你的筆電（一般規格）:
  - RAM：16 GB
  - SSD：512 GB
```

## 3. CPU：電腦的大腦

### 3.1 CPU 是什麼？

**CPU**（Central Processing Unit，中央處理器）= 電腦的「大腦」

```
┌─────────────────────────────────────────┐
│              CPU 內部結構                 │
│                                          │
│  ┌────────────────────────────────────┐ │
│  │        控制單元 (Control Unit)      │ │
│  │  ┌──────────────────────────────┐  │ │
│  │  │  • 讀取指令                    │  │ │
│  │  │  • 解碼指令                    │  │ │
│  │  │  │  • 協調各單元                │  │ │
│  │  └──────────────────────────────┘  │ │
│  └────────────────────────────────────┘ │
│               ↓         ↑                │
│  ┌────────────────────────────────────┐ │
│  │      運算單元 (ALU)                  │ │
│  │  ┌──────────────────────────────┐  │ │
│  │  │  • 加法、減法                  │  │ │
│  │  │  • 乘法、除法                  │  │ │
│  │  │  • 邏輯運算 (AND, OR, XOR)     │  │ │
│  │  │  • 比較運算                    │  │ │
│  │  └──────────────────────────────┘  │ │
│  └────────────────────────────────────┘ │
│               ↓         ↑                │
│  ┌────────────────────────────────────┐ │
│  │      暫存器 (Registers)              │ │
│  │  ┌──────┬──────┬──────┬──────┐     │ │
│  │  │  R0  │  R1  │  R2  │  R3  │ ... │ │
│  │  └──────┴──────┴──────┴──────┘     │ │
│  │  超高速暫存區（儲存運算中的數據）      │ │
│  └────────────────────────────────────┘ │
│                                          │
└─────────────────────────────────────────┘
```

---

### 3.2 CPU 如何工作？（取指-解碼-執行循環）

```
┌────────────────────────────────────────────────────────┐
│           CPU 的工作循環 (Fetch-Decode-Execute)          │
│                                                          │
│                                                          │
│  步驟 1: 取指 (Fetch)                                    │
│  ┌────────────────────────────────────┐                 │
│  │ 從記憶體讀取下一條指令                │                 │
│  │                                    │                 │
│  │  記憶體位址 0x100:  ADD R1, R2, R3 │                 │
│  │            ↓                       │                 │
│  │       讀取指令到 CPU                │                 │
│  └────────────────────────────────────┘                 │
│              ↓                                           │
│                                                          │
│  步驟 2: 解碼 (Decode)                                   │
│  ┌────────────────────────────────────┐                 │
│  │ 解析指令的意義                       │                 │
│  │                                    │                 │
│  │  ADD R1, R2, R3                    │                 │
│  │   ↓    ↓   ↓   ↓                   │                 │
│  │  運算 目的 來源1 來源2               │                 │
│  │  加法  R1   R2   R3                │                 │
│  │                                    │                 │
│  │  意思：R1 = R2 + R3                │                 │
│  └────────────────────────────────────┘                 │
│              ↓                                           │
│                                                          │
│  步驟 3: 執行 (Execute)                                  │
│  ┌────────────────────────────────────┐                 │
│  │ ALU 進行實際運算                    │                 │
│  │                                    │                 │
│  │  R2 = 5                            │                 │
│  │  R3 = 3                            │                 │
│  │   ↓                                │                 │
│  │  ALU 計算: 5 + 3 = 8               │                 │
│  │   ↓                                │                 │
│  │  R1 = 8 (結果寫回暫存器)            │                 │
│  └────────────────────────────────────┘                 │
│              ↓                                           │
│                                                          │
│  步驟 4: 回到步驟 1（處理下一條指令）                      │
│              ↓                                           │
│         無限循環 ♻️                                       │
│                                                          │
│  ┌──────────────────────────────────────┐               │
│  │  一個現代 CPU 每秒可執行數十億次循環   │               │
│  │  ARM Cortex-M3: 幾十 MHz ~ 100+ MHz  │               │
│  │  (每秒數千萬到上億次循環)              │               │
│  └──────────────────────────────────────┘               │
└────────────────────────────────────────────────────────┘
```

### 3.3 時脈頻率（Clock Speed）

CPU 的工作節奏由「時脈」控制：

```
┌────────────────────────────────────────┐
│            時脈信號                      │
│                                         │
│   ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐       │
│   │  │  │  │  │  │  │  │  │  │       │
│ ──┘  └──┘  └──┘  └──┘  └──┘  └──     │
│   ↑      ↑      ↑      ↑      ↑       │
│  Tick  Tick  Tick  Tick  Tick          │
│                                         │
│  每個 Tick = 1 個時脈週期                │
│  CPU 在每個週期執行部分指令               │
│                                         │
├────────────────────────────────────────┤
│  時脈頻率範例：                           │
│                                         │
│  50 MHz = 每秒 50,000,000 個週期        │
│  (本專案 LM3S6965 的典型時脈)            │
│                                         │
│  1 GHz = 每秒 1,000,000,000 個週期      │
│  (現代桌上型電腦 CPU)                    │
│                                         │
│  ┌──────────────────────────────────┐  │
│  │  越高的時脈 = 越快的運算速度       │  │
│  │  但也 = 越高的耗電量              │  │
│  └──────────────────────────────────┘  │
└────────────────────────────────────────┘
```

## 4. 記憶體：電腦的倉庫

### 4.1 記憶體階層

記憶體就像「倉庫」，用來儲存資料和程式。不同的記憶體有不同的速度和容量：

```
┌──────────────────────────────────────────────────────────┐
│                   記憶體金字塔                              │
│                                                            │
│                      ▲ 速度                                │
│                      │                                     │
│                                                            │
│                   ┌──────┐                                │
│                   │暫存器 │  最快，最小                      │
│                   │ CPU  │  幾十個，每個 32-bit             │
│                   │內建  │  存取時間：< 1 ns               │
│                   └──────┘                                │
│                  /        \                                │
│             ┌──────────────┐                              │
│             │  快取記憶體    │  很快，小                     │
│             │   (Cache)    │  幾 KB ~ 幾 MB               │
│             │  L1, L2, L3  │  存取時間：1-10 ns            │
│             └──────────────┘                              │
│            /                \                              │
│       ┌───────────────────────┐                           │
│       │   主記憶體 (RAM)        │  快，中等                  │
│       │ Random Access Memory  │  幾 MB ~ 幾 GB            │
│       │   (SRAM, DRAM)        │  存取時間：50-100 ns       │
│       └───────────────────────┘  ← LM3S6965: 64 KB       │
│      /                          \                          │
│  ┌──────────────────────────────────┐                     │
│  │  永久儲存記憶體 (Flash, ROM)       │  慢，大             │
│  │  Flash Memory / SSD / Hard Disk  │  幾 GB ~ 幾 TB      │
│  │         (非揮發性)                 │  存取時間：us ~ ms  │
│  └──────────────────────────────────┘  ← LM3S6965: 256KB │
│                                                            │
│                      │                                     │
│                      ▼ 容量                                │
│                                                            │
├───────────────────────────────────────────────────────────┤
│  名詞解釋：                                                  │
│  • ns (奈秒) = 10⁻⁹ 秒                                     │
│  • us (微秒) = 10⁻⁶ 秒                                     │
│  • ms (毫秒) = 10⁻³ 秒                                     │
│  • 揮發性：斷電後資料消失 (RAM, 暫存器)                       │
│  • 非揮發性：斷電後資料保留 (Flash, ROM, SSD)                │
└───────────────────────────────────────────────────────────┘
```

---

### 4.2 RAM vs Flash (ROM)

本專案的 LM3S6965 有兩種主要記憶體：

```
┌────────────────────────────────────────────────────────────┐
│                RAM vs Flash 比較                            │
├────────────────────────────────────────────────────────────┤
│                                                              │
│  RAM (Random Access Memory) - 隨機存取記憶體                 │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━                        │
│                                                              │
│  特性：                                                      │
│  ✓ 可讀可寫，速度快                                           │
│  ✓ 揮發性（斷電資料消失）                                      │
│  ✓ 用途：存放程式執行時的變數、堆疊                             │
│                                                              │
│  ┌────────────┐                                             │
│  │   RAM      │  電源 ON → 可讀寫                            │
│  │  0x2000... │     ↓                                       │
│  │            │  電源 OFF → 資料消失 ❌                       │
│  └────────────┘                                             │
│                                                              │
│  LM3S6965: 64 KB SRAM                                       │
│                                                              │
├────────────────────────────────────────────────────────────┤
│                                                              │
│  Flash (ROM) - 快閃記憶體                                    │
│  ━━━━━━━━━━━━━━━━━━━━━━                                     │
│                                                              │
│  特性：                                                      │
│  ✓ 可讀，寫入較慢                                             │
│  ✓ 非揮發性（斷電資料保留）                                    │
│  ✓ 用途：存放程式碼、常數                                      │
│                                                              │
│  ┌────────────┐                                             │
│  │   Flash    │  電源 ON → 可讀                              │
│  │  0x0000... │     ↓                                       │
│  │  程式碼    │  電源 OFF → 資料保留 ✅                        │
│  └────────────┘                                             │
│                                                              │
│  LM3S6965: 256 KB Flash                                     │
│                                                              │
└────────────────────────────────────────────────────────────┘

比喻：
  Flash = 硬碟（書架上的書，永久保存）
  RAM   = 書桌（工作區，關燈後清空）
```

### 4.3 記憶體位址

每個記憶體位置都有一個「位址」（Address），就像門牌號碼：

```
┌──────────────────────────────────────────────┐
│            記憶體位址示意圖                     │
│                                               │
│  位址          內容（1 Byte）                  │
│  (Address)    (Data)                          │
│  ────────     ──────                          │
│  0x00000000   10110101  ← Flash 起始位址       │
│  0x00000001   01001110                        │
│  0x00000002   11100011                        │
│  0x00000003   00110101                        │
│  ...          ...                             │
│  0x0003FFFF   10101010  ← Flash 結束位址       │
│               (256 KB)                        │
│                                               │
│  0x20000000   01010101  ← RAM 起始位址         │
│  0x20000001   11001100                        │
│  0x20000002   00111001                        │
│  0x20000003   10100110                        │
│  ...          ...                             │
│  0x2000FFFF   11110000  ← RAM 結束位址         │
│               (64 KB)                         │
│                                               │
│  ┌─────────────────────────────────────────┐ │
│  │  位址格式：0x 開頭代表十六進位            │ │
│  │  0x20000000 = 地址在 RAM 區域            │ │
│  │  0x00000000 = 地址在 Flash 區域          │ │
│  └─────────────────────────────────────────┘ │
└──────────────────────────────────────────────┘
```

## 📝 實作練習

### 練習 1：數字轉換

請完成以下轉換：

1. 二進位 `11010110` 轉成十進位 = ?
2. 十進位 `42` 轉成二進位 = ?
3. 十六進位 `0xFF` 轉成十進位 = ?
4. 十進位 `256` 轉成十六進位 = ?

<details>
<summary>解答</summary>

1. `11010110₂` = 128+64+16+4+2 = `214₁₀`
2. `42₁₀` = `101010₂`
3. `0xFF` = 15×16 + 15 = `255₁₀`
4. `256₁₀` = `0x100`

</details>

---

### 練習 2：記憶體計算

1. 64 KB = 多少 Bytes？
2. 256 KB = 多少 Bytes？
3. 如果 RAM 從 `0x20000000` 開始，大小為 64 KB，結束位址是？

<details>
<summary>解答</summary>

1. 64 KB = 64 × 1024 = `65,536 Bytes`
2. 256 KB = 256 × 1024 = `262,144 Bytes`
3. 起始：`0x20000000`
   大小：64 KB = 0x10000
   結束：`0x20000000 + 0x10000 - 1 = 0x2000FFFF`

</details>
